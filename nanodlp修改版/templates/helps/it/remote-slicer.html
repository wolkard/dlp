A causa di vincoli di prestazioni su alcune schede di sviluppo come Raspberry Pi, l'affettatore a distanza potrebbe essere sostanzialmente utilizzato per accelerare il processo e gestire file 3D più grandi.

Questa funzione delega il taglio a un'altra installazione di NanoDLP. Se questo campo non &eacuto riempito, il computer che carica una nuova piastra verrà sottoposto a scansione per l'installazione attiva di NanoDLP (solo sulla porta 8080) e verrà utilizzata come affettatrice predefinita per i sistemi ARM. Se la porta TCP IP: è stata specificata per il NanoDLP remoto, il sistema cercherà di utilizzarlo come affettatore a distanza.

Come funziona Slicer remoto: (esempio)

  1. NanoDLP è installato sul Raspberry Pi e sul computer Windows utilizzato dall'utente.
  2. Una nuova piastra viene caricata sul Raspberry Pi.
  3. Il Raspberry Pi con NanoDLP rileva l'installazione attiva di NanoDLP nel computer Windows dell'utente.
  4. Il Raspberry Pi trasferisce i dati richiesti a Windows.
  5. NanoDLP su Windows avvia il processo.
  6. NanoDLP su Raspberry Pi ottiene l'aggiornamento dello stato dalla versione di Windows. E aspetta che l'affettutura venga completata.
  7. NanoDLP su Raspberry Pi raccoglie i dati preparati.
